# SafeTea Mobile App — Technical Specification

## 1. Overview

SafeTea is a privacy-first dating safety platform where women anonymously share experiences about people they're dating. The mobile app is the primary product — the landing page drives downloads.

**Platform:** iOS + Android (single codebase)
**Stack:** React Native + Expo (SDK 52+)
**Backend:** Node.js + Express (or Fastify)
**Database:** PostgreSQL (primary) + Redis (cache/sessions)
**Auth:** Firebase Auth (phone/email) + custom JWT layer
**Storage:** AWS S3 / Cloudflare R2 (media)
**Search:** Meilisearch or Typesense (fast, typo-tolerant)
**Push Notifications:** Expo Push + Firebase Cloud Messaging
**Deployment:** Railway / Render (API) + Expo EAS (builds)

## 2. Core Features

### 2.1 Identity System

**Pseudonym Generator**
Every user gets or creates a pseudonym — never a real name.

- **Auto-Generate:** Combines a random adjective + noun + optional number
-   - Pool: ~200 adjectives (Brave, Quiet, Cosmic, Velvet, Neon, etc.)
    -   - Pool: ~200 nouns (Orchid, Falcon, Cipher, Ember, Sage, etc.)
        -   - Examples: "VelvetOrchid", "NeonCipher42", "CosmicSage"
            - - **Custom Create:** User types their own pseudonym
              -   - Min 3 chars, max 20 chars
                  -   - No real names allowed (checked against common name database)
                      -   - No slurs/profanity (filtered via profanity list)
                          -   - Must be unique across the platform
                              - - **Regenerate:** Users can regenerate up to 3x during onboarding, 1x per 30 days after
                                - - **Display:** Pseudonym shown everywhere — posts, comments, profile
                                 
                                  - **Avatar System**
                                  - Users select from pre-designed illustrated avatars (no photo uploads for privacy).
                                 
                                  - - **Categories (8-10 per category):**
                                    -   - Minimalist Silhouettes (various hairstyles, head shapes)
                                        -   - Abstract Shapes (geometric, colorful, artistic)
                                            -   - Nature Icons (flowers, animals, celestial)
                                                -   - Tea-Themed (teacups, tea leaves, kettles — on brand)
                                                    -   - Emoji-Style (expressive faces, moods)
                                                        - - **Customization:**
                                                          -   - Background color picker (12 preset colors)
                                                              -   - Optional border/frame (5 styles: none, circle, hexagon, rounded square, diamond)
                                                                  - - **Unlock System:**
                                                                    -   - 20 avatars free at signup
                                                                        -   - 30+ unlock at SafeTea+ subscription
                                                                            -   - 10 "rare" avatars earned via community contributions (50+ helpful posts)

                                                                                ```
                                                                                Screen 1: Welcome → "Date Smarter. Stay Safer."
                                                                                Screen 2: Phone/Email Verification (Firebase Auth)
                                                                                Screen 3: City Selection (from active cities + vote for new)
                                                                                Screen 4: Pseudonym (Generate or Create)
                                                                                Screen 5: Avatar Selection (pick + customize)
                                                                                Screen 6: Community Guidelines Agreement
                                                                                Screen 7: Home Feed
                                                                                ```

                                                                                **Verification Requirements (all users):**
                                                                                - Valid phone number (SMS code) OR email verification
                                                                                - - Phone preferred — harder to create fake accounts
                                                                                  - - Rate limit: max 2 accounts per phone number
                                                                                    - - Flagged carriers/VoIP numbers blocked (Twilio Lookup API)

                                                                                      ### 2.3 Moderator Approval Process (Stringent)

                                                                                      Moderators have elevated power (remove posts, issue warnings, ban users). The approval process must be rigorous.

                                                                                      **Eligibility Requirements (must meet ALL):**
                                                                                      1. Account age: minimum 90 days active
                                                                                      2. 2. Community standing: 0 active strikes, no bans in history
                                                                                         3. 3. Contribution score: minimum 50 posts + 100 comments with positive engagement
                                                                                            4. 4. Report accuracy: if user has submitted reports, 70%+ must have been valid
                                                                                               5. 5. City residency: must be in the city they want to moderate for 30+ days
                                                                                                 
                                                                                                  6. **Application Process (5 steps):**
                                                                                                 
                                                                                                  7. ```
                                                                                                     Step 1: Intent Form
                                                                                                       - Why do you want to moderate?
                                                                                                       - What does safety mean to you? (min 100 chars)
                                                                                                       - How many hours/week can you commit?
                                                                                                       - Which city are you applying for?

                                                                                                     Step 2: Scenario Assessment (auto-scored)
                                                                                                       - 5 real-world moderation scenarios presented
                                                                                                       - User selects best action for each
                                                                                                       - Must score 4/5 or higher to proceed
                                                                                                       - Example: "A post names someone directly and includes their workplace. What do you do?"
                                                                                                         a) Remove immediately — violates privacy policy ✓
                                                                                                         b) Leave it up — it's free speech
                                                                                                         c) Edit the post to remove the name
                                                                                                         d) Report to admin only

                                                                                                     Step 3: Identity Verification
                                                                                                       - Government ID upload (encrypted, reviewed by admin only, deleted after verification)
                                                                                                       - ID must match phone number region
                                                                                                       - Purpose: prevent bad actors from moderating
                                                                                                       - Stored: only a verified/not-verified flag — ID image purged within 72 hours

                                                                                                     Step 4: Admin Review
                                                                                                       - SafeTea admin manually reviews application
                                                                                                       - Checks user history, scenario score, and written responses
                                                                                                       - Admin can approve, reject, or request more info
                                                                                                       - Target turnaround: 48-72 hours

                                                                                                     Step 5: Probationary Period (30 days)
                                                                                                       - New moderator gets "Junior Mod" badge
                                                                                                       - Actions are reviewed by a Senior Mod weekly
                                                                                                       - Limited powers: can flag but not remove (Senior Mod confirms)
                                                                                                       - After 30 days: auto-promoted to full Mod if no issues
                                                                                                       - If issues found: probation extended 30 days or revoked
                                                                                                     ```
                                                                                                     
                                                                                                     **Moderator Tiers:**
                                                                                                     | Tier | Title | Powers | Requirement |
                                                                                                     |------|-------|--------|-------------|
                                                                                                     | 1 | Junior Mod | Flag content, mute users (1hr) | Pass application |
                                                                                                     | 2 | Moderator | Remove posts, warn users, mute (24hr) | 30 days clean probation |
                                                                                                     | 3 | Senior Mod | Ban users (temp), review Junior Mod actions, approve new mods | 6 months + admin nomination |
                                                                                                     | 4 | City Lead | All mod powers + manage mod team for their city, access analytics | 12 months + admin appointment |
                                                                                                     
                                                                                                     **Mod Accountability:**
                                                                                                     - All mod actions are logged with timestamps
                                                                                                     - - Users can appeal any mod action (goes to Senior Mod or admin)
                                                                                                       - - Quarterly review: mods with <80% appeal-upheld rate get retrained or removed
                                                                                                         - - Abuse of power = immediate removal + permanent mod ban

                                                                                                           ### 2.4 Post System
                                                                                                           
                                                                                                           **Post Types:**
                                                                                                           1. **Experience Post** — Share an experience about someone you dated
                                                                                                           2.    - Text only (no photos of people — privacy)
                                                                                                                 -    - First name + first initial of last name only (e.g., "Jake M.")
                                                                                                                      -    - Dating app where you met (Hinge, Bumble, Tinder, etc.)
                                                                                                                           -    - City/neighborhood
                                                                                                                                -    - Date range (approximate)
                                                                                                                                     -    - Categories: Red Flag, Green Flag, PSA, Question
                                                                                                                                          -    - Optional: verified by other members
                                                                                                                                           
                                                                                                                                               - 2. **Community Post** — General safety discussion
                                                                                                                                                 3.    - Dating tips, safety advice, app reviews
                                                                                                                                                       -    - No specific person named
                                                                                                                                                        
                                                                                                                                                            - 3. **Alert Post** (Mods only) — Safety alerts
                                                                                                                                                              4.    - Crime patterns, scam alerts, amber-style warnings
                                                                                                                                                                    -    - Pinned to top of city feed
                                                                                                                                                                     
                                                                                                                                                                         - **Post Rules:**
                                                                                                                                                                         - - No full names, no photos, no workplace addresses
                                                                                                                                                                           - - No revenge content — must be factual experience
                                                                                                                                                                             - - Character limit: 500 chars for posts, 250 for comments
                                                                                                                                                                               - - Posts auto-expire after 12 months (archived, not deleted)
                                                                                                                                                                                 - - Edit window: 15 minutes after posting
                                                                                                                                                                                  
                                                                                                                                                                                   - ---
                                                                                                                                                                                   
                                                                                                                                                                                   ### 2.5 Search & Matching
                                                                                                                                                                                   
                                                                                                                                                                                   Users search by first name + city to find if anyone has posted about someone.
                                                                                                                                                                                   
                                                                                                                                                                                   - **Search input:** First name, optional last initial, city
                                                                                                                                                                                   - - **Fuzzy matching:** handles typos (e.g., "Jonh" → "John")
                                                                                                                                                                                     - - **Results:** grouped by person (matched by name + city + approximate details)
                                                                                                                                                                                       - - **Privacy:** searcher identity never revealed to poster

Security Considerations
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - Phone numbers are hashed (SHA-256 + salt) — never stored in plain text
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - All API calls over HTTPS
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - JWT tokens with 15-min expiry + refresh tokens (7 days)
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - Rate limiting on all endpoints (express-rate-limit)
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - Input sanitization (DOMPurify equivalent for text)
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - Mod ID verification images encrypted at rest, auto-deleted after 72 hours
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - GDPR/CCPA compliance: users can export or delete all their data
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - No IP logging beyond rate limiting
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               - Content moderation AI layer (optional Phase 6): OpenAI moderation API for auto-flagging
